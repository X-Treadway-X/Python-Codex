Welcome to Python 3 Learning

Python is a powerful, easy-to-learn language. These lessons will guide you through the basics and build a strong foundation.

Lesson 1: Your First Python Code

Starting with the famous “Hello, World!” program. It's the simplest way to run Python code.

print("Hello, world!")

Try changing the text inside the quotes and run it again.

Lesson 2: Variables

Variables hold data. You can store text, numbers, and more.

name = "Alice"
age = 25
print(name)
print(age)

Lesson 3: Comments

Comments help explain your code and are ignored by Python.

# This is a comment
print("This will run")

Lesson 4: Data Types

Python has different data types to store different kinds of data.

x = 10        # int
pi = 3.14      # float
name = "Bob"   # str
is_active = True  # bool
numbers = [1, 2, 3]  # list

Lesson 5: If / Else Statements

Control your program flow by making decisions.

age = 20
if age >= 18:
    print("You are an adult.")
else:
    print("You are a minor.")

Lesson 6: Loops

Repeat tasks efficiently with loops.

# For loop
for i in range(5):
    print(i)

# While loop
count = 0
while count < 5:
    print(count)
    count += 1

Lesson 7: Functions

Functions let you reuse code and organize your program.

def greet(name):
    print(f"Hello, {name}!")

greet("Alice")
greet("Bob")

Lesson 8: Lists and List Methods

Lists store multiple items and have useful methods.

fruits = ["apple", "banana", "cherry"]
print(fruits)
fruits.append("orange")  # add item
print(fruits)
print(fruits[1])  # banana

Lesson 9: Dictionaries

Dictionaries store key-value pairs for fast lookup.

person = {"name": "Alice", "age": 30}
print(person)
print(person["name"])  # Alice
person["age"] = 31  # update value

Lesson 10: Tuples and Sets

Tuples are immutable lists, sets store unique items.

point = (10, 20)  # tuple
print(point)

unique_numbers = {1, 2, 2, 3}
print(unique_numbers)  # {1, 2, 3}

Lesson 11: User Input

Get input from the user during program execution.

name = input("Enter your name: ")
print(f"Hello, {name}!")

Lesson 12: Exception Handling

Handle errors gracefully to prevent crashes.

try:
    x = int(input("Enter a number: "))
    print(10 / x)
except ZeroDivisionError:
    print("Cannot divide by zero.")
except ValueError:
    print("Invalid input.")

Lesson 13: Modules and Imports

Use external code libraries with import statements.

import math
print(math.sqrt(16))

Lesson 14: File I/O Basics

Read from and write to files to store data.

# Write to a file
with open("test.txt", "w") as file:
    file.write("Hello, file!")

# Read from a file
with open("test.txt", "r") as file:
    content = file.read()
    print(content)

Lesson 15: List Comprehensions

A concise way to create lists:

squares = [x**2 for x in range(10)]
print(squares)

Lesson 16: Lambda Functions

Anonymous, short functions:

add = lambda a, b: a + b
print(add(5, 3))  # 8

Lesson 17: Classes and Objects

Object-oriented programming basics:

class Dog:
    def __init__(self, name):
        self.name = name

    def bark(self):
        print(f"{self.name} says Woof!")

my_dog = Dog("Rex")
my_dog.bark()

Lesson 18: Decorators (Intro)

Modify functions easily:

def decorator(func):
    def wrapper():
        print("Before function")
        func()
        print("After function")
    return wrapper

@decorator
def say_hello():
    print("Hello!")

say_hello()

Lesson 19: Generators

Functions that yield values lazily:

def count_up_to(n):
    count = 1
    while count <= n:
        yield count
        count += 1

for number in count_up_to(5):
    print(number)

Lesson 20: Useful Built-in Functions

Python includes many handy built-ins:

  • len() - length of an object
  • range() - generate number sequences
  • type() - check variable type
  • isinstance() - check class membership
  • enumerate() - get index and value when looping

Lesson 21: Virtual Environments (Best Practice)

Keep project dependencies isolated:

python3 -m venv myenv
source myenv/bin/activate  # On Windows: myenv\Scripts\activate
pip install package_name
deactivate

Using virtual environments helps avoid package conflicts between projects.
